# CLAUDE.md — Meteor Idle

## What this is

A 2D Unity 6 idle game. Voxel meteors fall; the bottom row has 4 base slots (weapons), 2 drone bays, and 1 collector (rock grinder). Weapons auto-fire to destroy meteor voxels. Core voxels spawn CoreDrop entities that collector drones retrieve and deliver to the Collector for cash. Two weapon types (Missile and Railgun), collector drones with custom physics. Desktop/laptop 16:9. Early development — no persistence, no audio.

## Tech stack

- Unity **6000.4.1f1** (pinned)
- 2D URP, new Input System, TextMeshPro
- C# in `MeteorIdle` assembly (`Assets/Scripts/MeteorIdle.asmdef`)
- Tests: `Assets/Tests/EditMode/` (EditMode) and `Assets/Tests/PlayMode/` (PlayMode)
- Unity MCP (`mcpforunity://` resources, `mcp__UnityMCP__*` tools)

## Conventions

**Testing:** EditMode tests for pure logic. Update/add tests when changing tested modules. Run via `mcp__UnityMCP__run_tests`. Zero failures before committing.

**Final verification gate:** Local WebGL dev build, not Unity Editor play mode. Editor play is fine for iterative debugging but not sign-off.

**Fast iterative loop (in-editor):**
1. Edit C# / `execute_code` -> `refresh_unity` -> `read_console` (zero errors)
2. `run_tests` if touching tested logic
3. `manage_editor play` -> `manage_camera screenshot` -> `read_console` -> `manage_editor stop`

**Final verification loop (WebGL dev build):**
1. `execute_code` calling `BuildScripts.BuildWebGLDev()` (editor stays open, builds in-process)
2. `tools/serve-webgl-dev.sh` in background
3. `chrome-devtools-mcp`: navigate `localhost:8000`, exercise the change, check console, screenshot
4. Close tab, kill server

**Procedural art only.** Every PNG generated by `execute_code` + `Texture2D.EncodeToPNG()`. Voxel aesthetic: per-pixel SetPixel, 1-px dark edges, ~20% brighten highlights. No smooth lines, gradients, or LineRenderers.

**Branch workflow.** Work on `iter/<name>`, never push main directly. Fast-forward main only after user verification.

**Identity rules.** Repo-local git config only — never touch global. Remote uses SSH alias `github-muwamath`. No `Co-Authored-By:` trailers. Never use `gh` CLI on this repo. Run `python3 tools/identity-scrub.py` before every commit.

**Spawner tuning.** `initialInterval=12s`, `minInterval=4.5s`, `rampDurationSeconds=180s`. Deliberately calm — don't regress.

## Unity MCP gotchas

- `execute_code` runs C# 6 (CodeDom). No `using` inside the body — use fully-qualified types.
- `Object` is ambiguous: always write `UnityEngine.Object.DestroyImmediate(...)`.
- Don't call `MarkSceneDirty`/`SaveScene` during play mode.
- `manage_camera screenshot` excludes Screen Space Overlay canvases. Verify UI via `execute_code` CanvasGroup checks.
- `PlayerSettings.runInBackground = true` must stay on.

## Debug overlay

Editor-only, backquote toggle, pauses via `Time.timeScale = 0f`. Extend the existing `DebugOverlay.cs` — don't create new debug systems.

## Before committing / pushing

- `git status` — only expected files
- `python3 tools/identity-scrub.py` — staged diff (before commit) and `main..HEAD` (before push)
- `git log --format="%an <%ae>" -5` — only repo-local identity
- No force-push main without explicit permission
- Work on a branch, branch names: `iter/<kebab-description>`

## Before promoting a branch to `main`

1. Both test suites green (EditMode + PlayMode)
2. WebGL dev build verification (see final verification loop above)
3. Identity scrub clean
4. User sign-off
5. After fast-forward: prod WebGL build (`BuildScripts.BuildWebGL()`) -> `tools/deploy-webgl.sh` -> push gh-pages -> verify live site
